Abstract

Lost in my code, I didn't realize until after the last release than Squirrel was not all that easy to get started with, The Developer's Guide was not entirely clear. After receiving some feedback from brave and confused users, I decided to write a short document, in the opes to clarify key points and help people get started in the language Squirrel.

1. The Squirrel distribution

You have just installed Squirrel in your BeOS computer and you anxious to try it, but you don't know how to start...

The best things to do, is to try out the various demos available with the Squirrel distribution. Squirrel is installed in /boot/apps/Squirrel. You will find all the files in this directory, but the Add-Ons are located in /boot/home/config/add-ons/Squirrel. You will not be interacting with those files directly, so you may ignore them for the moment.

You're encouraged to read the ReadMe file which contains useful informations about the release. As well, reading the Developer's Guide is a mandatory step for a serious start in Squirrel. You will find an HTML version in the Developer's Guide directory as well as a PDF version: Developer's Guide.pdf. If you going to play with Skippy you may read the Skippy's User Manual.pdf.

The folder GUI Tutorial contains two articles published in December '99 and January '00 in BeGroovy. They both deal with Graphical User Interface building using Squirrel. If you're interested in GUI, you will need to read the GUI Manual.pdf PDF file (there's no HTML version available for now).

2. Running the Demos

The folder Demos contains severals demonstration scripts. Each of these is reconized as a Squirrel script file by it MIME type: text/x-script.squirrel. Now, to run a file we could :

  • use a Terminal to execute it like any other executable
  • load it from the Squirrel console
  • set the preferred application to Squirrel
Setting Squirrel as the Preferred Application is the simplest and faster way to run a script file. This may, of course, vary from each script.

For example, if we consider the file GUI Demo.sqi, once the Preferred Application has been set to Squirrel.dr4.7, a double click on the script's icon will launch it like any other application:

3. The console
As we have seen earlier, launching Squirrel's Console.dr4.7 start Squirrel with a console, but what is a console ? What is it good for ?

The console act the same way as the BeOS Terminal, or any other Unix-like (or Dos-like) terminal. It display a prompt, waits for the user's inputs and then processes the command. It's useful for those who want to discover Squirrel interactively, by submitting orders and then observing the results.

The Squirrel's console also allow you also to load and execute scripts like the demos by the menu Script. Some settings can also be changed.

4. Writing and running your own script

Want to create your first Squirrel script ? Easy! What do you need ?

  • an idea :)
  • a text file editor
  • a Squirrel interpreter of course :)
The choice of the text editor is up to you. By default, BeOS is shipped with StyledEdit and of course the famous (and infamous) vi. Personnaly, I use (and prefer) Pe, but that's a matter of personal tast (even if Pe is great :).

5. Squirrel scripts like shell scripts

You may want to execute a Squirrel script like any shell script or any other executable from the BeOS Terminal.

It's easy to execute it like that. You just have to add to the first line of your script :

#!/boot/apps/Squirrel/Squirrel.dr4.7

Of course, you could have also used Squirrel's Console.dr4.7 to execute your script file in the Squirrel's console rather than the terminal.

Inputs and outputs occur then through the BeOS Terminal.
